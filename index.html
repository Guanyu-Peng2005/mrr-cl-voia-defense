01 · 项目背景与研究意义

从完整上报优先，
转向风险相关价值优先。

多无人机追捕中，子机持续产生本地观测，母机需要融合轨迹并分配任务。但真实链路存在带宽、时延、抖动、丢包和乱序，盲目全量上报会挤占通信资源，固定门控又可能在风险最高的时刻错过关键证据。

任务压力 高速目标 + 障碍环境

追捕状态快速变化，过期信息会直接影响任务分配。

通信压力 带宽限制 + 丢包乱序

全量通信并不等于可靠，低价值包会增加链路拥塞。

融合压力 NIS 一致性 + 安全约束

错误融合会传导到控制层，造成追捕失败或安全违规。

从 多发数据
转向 发送有价值的信息
最终服务 安全可靠追捕
02 · 研究内容与算法框架

闭环准入：源端选包，收端验信，安全回退。

MRR-CL-VoIA 不是单一阈值，也不是固定比例限流。它把任务风险、信息价值、通信代价和收端一致性放进同一闭环，使有限带宽优先服务高风险、高价值、高可信的信息。

源端决策

价值驱动上报

子机根据风险状态、信息增益、新鲜度、置信度和通信代价计算单位字节价值，选择 full report、compact report、heartbeat 或 suppress。

输入：本地估计、协方差、观测质量 输出：按价值选择的数据包 目的：避免无差别全量上报
03 · 算法原理

参数由风险归一化、信息增益和通信代价共同约束。

价值函数由风险状态归一化、信息增益、通信代价和一致性准入共同决定。核心不是某个固定权重，而是“风险越高、信息越新、越可信，越应该发；通信越贵、越低价值，越应该压缩或抑制”。

01

源端 VoI 评分

计算单位字节价值，让数据包先在源端被筛选。

J = ΔU / C
  • ΔU：预期不确定度下降与任务风险收益。
  • C：完整包、紧凑包、心跳包的通信代价。
02

收端 NIS 准入

母机不直接融合所有包，而是先判断创新是否一致。

ε = νTS-1ν
  • 一致且新鲜：接受并融合。
  • 可疑但有价值：降权保留。
  • 明显异常：拒绝进入融合。
03

安全哨兵回退

风险升高时主动进入保守策略，避免降低通信占用削弱任务安全。

rv, rr, rn → fallback
  • 高速、雷达退化、队伍资源紧张触发回退。
  • 低风险持续一段时间后恢复标准策略。

VoI 决策实验台

风险变量驱动的上报策略响应

通过风险状态、信息增益、新鲜度、置信度与通信代价的参数变化，呈现源端包型、链路占用和收端处理方式的联动关系，用于说明价值驱动上报与固定比例限流的差异。

VoI score 0.00 compact report

信息仍有价值，但通信代价需要控制，发送紧凑包。

源端：发送紧凑报告 链路：压缩证据、控制字节数 收端：准入后低成本融合
04 · 基线设计与公平对比

基线设计：把通信、准入与安全逐项拆解。

对比方法的设计目标，是把 MRR-CL-VoIA 的贡献拆成可检验的问题：只增加通信会怎样，只做一致性门控会怎样，只控制平均带宽会怎样，以及只用事件触发是否足够。

Baseline logic

四类对照分别固定一个关键假设

所有方法共享同一任务场景、压力单元、配对随机种子和评价指标。差异只放在上报策略、收端准入和安全回退机制上，使结果能够回答“本文方法的收益来自哪里”。

通信强度 上报选择 一致性准入 安全回退
Baseline 1

All-Full

所有子机在每个通信时刻发送完整状态、协方差和质量信息，用作高通信参考。

控制问题
检验“全量通信是否一定带来更高安全性”。
保留机制
收端仍执行一致性检查和融合，避免把它做成不公平的弱基线。
预期作用
给出通信开销上界，并暴露低价值或异常信息对融合的潜在干扰。
Baseline 2

Event-NIS

用 NIS 创新统计量触发上报或准入，强调“异常事件发生时才通信”。

控制问题
检验只依赖一致性事件触发，是否足以替代价值感知上报。
保留机制
使用同一状态估计和融合框架，但不引入通信代价归一化。
预期作用
说明 NIS 能发现异常，却不能完整表达任务风险和带宽约束。
Baseline 3

Budget-Relaxed Rand-Mix

按随机比例混合 full、compact 和 suppress，使平均带宽接近或略高于本文方法。

控制问题
检验性能提升是否只是来自平均通信量降低或包型比例变化。
保留机制
保持相近通信预算，但去掉风险、信息增益和安全哨兵闭环。
预期作用
若随机混合仍显著落后，说明关键在于价值驱动选择而非平均带宽。
Baseline 4

NIS-99.9

采用更严格的 99.9% NIS 门限，只让统计上一致性很强的信息进入融合。

控制问题
检验单纯收紧收端准入，能否替代源端价值筛选和安全回退。
保留机制
融合和任务分配保持一致，仅改变准入门限的保守程度。
预期作用
展示过强门控可能牺牲关键风险时刻的信息可用性。
同一压力矩阵

速度、丢包、雷达退化、障碍密度和队伍规模保持一致。

配对随机种子

每个方法在同一组随机场景上比较，降低偶然场景偏差。

同一核心指标

统一报告任务成功率、完全安全成功率和平均通信带宽。

05 · 项目成果与实验结果

主结论：在保持安全成功率的同时，把通信花在更该花的位置。

实验覆盖 21 个压力单元、每单元 30 个配对随机种子、5 种方法、每方法 630 个 episode。重点不是单点跑赢，而是同一压力矩阵下同时比较任务成功率、完全安全成功率和带宽。

21压力单元

速度、丢包、雷达退化、障碍密度、队伍规模。

30配对随机种子

配对随机种子，降低随机场景偏差。

5对比方法

MRR-CL-VoIA、Event-NIS、Rand-Mix、NIS-99.9、All-Full。

630每方法 episode

完整主实验统计和动态图生成。

为什么不是 All-Full？

全量上报通信更重，而且低价值或不一致证据可能干扰融合。本文方法在关键时刻保留信息，在非关键时刻节省链路。

为什么不是 Rand-Mix？

Rand-Mix 让包型比例和本文方法接近且带宽略高，但没有风险状态闭环，因此安全成功率仍然落后。

为什么不是固定 NIS？

固定门控无法根据雷达退化、高速逃逸和队伍压力自适应调整准入，容易在关键场景错过证据。

最终总结

MRR-CL-VoIA 的一句话价值

本方法使多无人机在有限通信下优先传输关键、新鲜、可信且安全相关的信息，并将其送入一致性准入与融合闭环。

高安全成功率 低通信带宽 闭环可解释 适合扩展到真实链路
